<h1>웹 랜딩 페이지 퍼블리싱 프로젝트<br>
Static Web Landing Page – Publishing Project</h1>

<h2>1. 프로젝트 개요</h2>
본 프로젝트는 HTML과 CSS를 기반으로 제작한 <strong>정적 웹사이트 구현 프로젝트</strong>로,
웹 표준을 준수한 시맨틱 마크업과 반응형 웹 설계를 통해 <strong>다양한 디바이스 환경에서 일관된 사용자 경험을 제공하는 것</strong>을 목적으로 개발하였습니다.

디자인 시안 구현에 그치지 않고, 실제 서비스 환경을 고려하여 유지보수성과 확장성을 갖춘 구조로 설계하였으며,
명확한 폴더 구조와 클래스 네이밍, 반응형 대응 전략을 적용하였습니다.
또한 GitHub Pages를 활용해 배포까지 완료하여, 실사용 가능한 결과물로 완성하였습니다.

<h2>5. 구현 포인트</h2>
<h3>1. 시맨틱 마크업 기반 구조 설계</h3>

header, main, section, footer 등 HTML5 시맨틱 태그를 적극 활용하여 문서 구조를 명확히 정의

검색 엔진 최적화(SEO)와 접근성을 고려한 마크업 구성

이미지 요소에 alt 속성을 적용하여 정보 전달력 강화

<h3>2. Flexbox를 활용한 레이아웃 구성</h3>

header, main, section, footer 등 시맨틱 태그를 활용하여

콘텐츠의 의미와 구조가 명확하게 드러나도록 구현

이를 통해 가독성과 유지보수성을 동시에 고려한 코드 구조를 유지

<h3>3. 반응형 웹 구현</h3>

Media Query를 활용하여 디바이스 해상도별 UI 대응

화면 크기에 따라 네비게이션 메뉴 구조 변경

데스크톱: 전체 메뉴 노출

모바일: 메뉴 숨김 및 아이콘 형태로 전환

이미지 및 텍스트 요소를 유동적으로 조정하여 화면 깨짐 현상 최소화

<h3>4. 고정 헤더(Fixed Header) 구현</h3>

position: fixed를 적용한 상단 헤더 구성

z-index 관리로 스크롤 시에도 다른 요소 위에 안정적으로 노출

고정 헤더로 인한 콘텐츠 가림 현상을 고려해 main 영역에 여백 적용
<hr />

<h2>6. 결론 및 느낀점</h2>
본 프로젝트를 통해 HTML과 CSS를 활용한 웹 페이지 구현의 전반적인 흐름을 경험하였으며,
단순한 화면 구성보다 <strong>구조 설계와 유지보수 관점의 중요성</strong>을 명확히 인식할 수 있었습니다.

특히 시맨틱 마크업을 기반으로 문서 구조를 설계하면서,
초기 단계에서의 구조적 판단이 이후 스타일링과 반응형 대응의 난이도를 크게 좌우한다는 점을 체감하였습니다.
의미 있는 태그 사용은 가독성과 접근성뿐 아니라, 코드 관리 측면에서도 효율적이라는 점을 확인할 수 있었습니다.

반응형 웹 구현 과정에서는 디바이스 해상도에 따라 레이아웃을 단순히 축소하는 것이 아니라,
<strong>콘텐츠 우선순위와 사용자 동선</strong>을 고려한 재배치가 중요하다는 점을 학습하였습니다.
이를 통해 다양한 환경에서도 안정적인 사용자 경험을 제공하는 설계 역량을 기를 수 있었습니다.

또한 GitHub를 활용한 버전 관리와 배포 과정을 직접 경험하며,
개발 결과물을 실제 서비스 형태로 제공하기 위해서는 구현뿐 아니라
<strong>관리, 배포, 검수까지 포함한 전체 프로세스에 대한 이해가 필요함</strong>을 인지하게 되었습니다.

본 프로젝트는 프론트엔드 개발의 기초 역량을 체계적으로 정리한 결과물로서,
향후 JavaScript 기반 인터랙션 추가 및 컴포넌트 단위 개발로 확장할 수 있는 기반이 되었습니다.
이를 바탕으로 실무 환경에서도 요구사항을 정확히 분석하고, 유지보수가 가능한 코드를 작성하는 개발자로 성장하고자 합니다.
